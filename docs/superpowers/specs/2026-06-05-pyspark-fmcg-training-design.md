# PySpark FMCG Training Kit — Design

**Date:** 2026-06-05
**Status:** Approved

## Goal

A self-contained PySpark training/assessment kit on an FMCG (fast-moving
consumer goods) theme. It includes a configurable data generator that produces
realistic, deliberately-dirty datasets, plus mid-level Jupyter exercises across
four topic areas. Each exercise notebook has hidden tasks, inline auto-check
cells (for objective grading), and a matching filled-in solution notebook (for
learning). Java is auto-installed so the kit runs with no manual JVM setup.

## Decisions

| Decision | Choice |
|----------|--------|
| Purpose | Both — exercises with auto-checks (grading) + solution notebooks (learning) |
| Level | Mid-level Data Engineer |
| Format | Jupyter notebooks |
| Coverage | Core DataFrame ops; Joins & windows; Data quality; Performance & advanced |
| Java | Auto-install portable Temurin JDK 17 into `.jdk/` (no sudo) |
| Data scale | Moderate (~100k–1M rows), generator parameterized by scale factor |
| Dirty data | Yes — inject realistic data-quality problems |

## FMCG data model (star schema)

| Dataset | Type | Key columns |
|---|---|---|
| `dim_products` | dimension | product_id, name, category, subcategory, brand, pack_size, unit_cost, supplier_id |
| `dim_stores` | dimension | store_id, name, region, city, channel (hyper/super/convenience/online), size_sqm, open_date |
| `dim_customers` | dimension | customer_id, name, loyalty_tier, signup_date, region |
| `dim_suppliers` | dimension | supplier_id, name, country, lead_time_days |
| `dim_promotions` | dimension | promo_id, promo_type, discount_pct, start_date, end_date |
| `fact_sales` | fact (large) | txn_id, date, store_id, product_id, customer_id?, qty, unit_price, discount, promo_id? |
| `fact_inventory` | fact | date, store_id, product_id, stock_on_hand, reorder_point |

### Injected data problems (seeded, reproducible)
- Nulls in `customer_id` / `promo_id`
- Duplicate transactions
- Inconsistent casing / whitespace in categories & regions
- Quantities stored as strings, some negative
- Out-of-range discount values
- Orphan foreign keys (sales referencing missing products)
- One deliberately skewed mega-store (for the skew exercise)

Raw/dirty sources are written as **CSV**; clean reference dimensions as
**Parquet**.

## Exercises (mid-level, ~5–6 tasks per notebook)

- **01 Core DataFrame ops** — load, select/filter, derived columns
  (revenue = qty·price·(1−disc)), groupBy/agg, sorting, top-N per category.
- **02 Joins & windows** — enrich sales with product/store dims, broadcast vs
  shuffle, rank top products per region, running 7-day sales, week-over-week
  growth (lag), customer RFM-style window.
- **03 Data quality** — detect & fix nulls, dedup transactions, normalize
  casing/whitespace, cast string→int safely, flag orphan FKs, validate discount
  ranges.
- **04 Performance & advanced** — partitioning & partition pruning, broadcast
  join tuning, diagnosing the skewed mega-store, caching, UDF vs native function
  comparison, equivalent Spark SQL.

## How "Both" works in notebooks

Each task = a markdown prompt + a stub cell (`# TODO`) + an **auto-check cell**
that `assert`s on the candidate's result (objective pass/fail for grading).
`notebooks/solutions/` holds the same notebooks with stubs filled in. Notebooks
are generated by `build_notebooks.py` so each task's prompt, check, and solution
are defined once and stay in sync.

## Dependencies to add

- `nbformat` — build notebooks programmatically
- `jupyterlab` — run the notebooks

Generation uses Spark + Python stdlib (seeded `random`, curated value lists) —
deterministic, no heavy extras.

## Java bootstrap

`scripts/bootstrap_java.py` downloads a portable Temurin JDK 17 tarball into
`.jdk/` (gitignored) and `utils/spark.py` points `JAVA_HOME` there automatically,
so `build_session()` works with no sudo.
